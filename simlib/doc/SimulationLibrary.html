Classes

MotorUnit
This class defines the location and fibres for a single motor unit. See also MuscleFibre and MuscleData.
Muap
This class controls a definition of a single Muap, and contains the interface to save/restore it to a file store.

This class is used within the Jitter code, as the individual MFAP's managed within this code are accessed in order to generate a Jitter'ed MUAP when required.

This class contains the data to generate a MUAP train. An instance of this object will contain a set of MFAP firings, which can be used with a given Jitter value to create a MUAP train.

See the calls:
int calcJitteredMuap()
and
muapDataElement *getCurrentMuap();
for details.

MuscleData
This class holds the physical definition of the current muscle. See also MotorUnit and MuscleFibre.
MuscleFibre
This class defines the location, motor unit, neural attachment point in Z (jShift) and diameter of a single muscle fibre See also MotorUnit and MuscleData.
NeedleInfo
This class defines the physical positioning of the needle.
SimulationResult
A wrapper for all the result data from a simulator run.

This class will contain all the resulting data for a simulation run including:

  • The EMG signal, via getEmgData()
  • Motor unit and muscle fibre data, via getMuscleData()
  • Muap structures, via getMuap()
Simulator
This object wraps up all the simulator functionality -- calling run on this object will return a result object which can be used to get all the data from a single simulator run.
